# NextOpsAgent Project Plan

## Product direction

NextOpsAgent is a local-first SME operations diagnosis project. It converts
informal workflow text into structured evidence, bottlenecks, automation
priorities, directional ROI assumptions, and an implementation roadmap.

## Stage plan

### NEXT-OPS-002 - Local Skeleton + Testable MVP Baseline

Status: **Complete**

- Establish modular Python architecture.
- Implement transparent rule-based diagnosis.
- Add fictional workflows and tests.
- Provide a simple Streamlit interface and Markdown export.

### NEXT-OPS-003 - Core Engine Upgrade

Status: **Complete**

- Improve phrase coverage, evidence quality, and category precision.
- Add configurable scoring and ROI inputs.
- Add workflow-level summary metrics and stronger confidence reasoning.
- Expand tests for ambiguity, empty input, and workflow edge cases.

### NEXT-OPS-004 - Streamlit UI Product Polish

Status: **Complete**

- Add a product-style Streamlit dashboard.
- Add hero section, metric cards, rounded cards, badges, and polished empty states.
- Improve sidebar demo controls and sample workflow loading.
- Present diagnosis, scores, ROI, roadmap, and report across five clean tabs.
- Update screenshot guidance and README showcase wording.

### NEXT-OPS-005 - Report Export & Showcase Assets

Status: **Complete**

- Add Markdown and HTML report export support.
- Add local demo report generation from fictional sample workflows.
- Add public release checklist and showcase manifest updates.
- Add portfolio folders for future screenshots and demo output previews.

### NEXT-OPS-006 - GitHub Public Showcase Release

Status: **Local release candidate**

- Initialize/review local Git state.
- Prepare small portfolio demo outputs from fictional workflows.
- Prepare a clean local commit.
- Capture real showcase screenshots before enabling README image links.
- Publish to GitHub only after explicit approval.

### NEXT-OPS-007 - GitHub Remote Setup + Public Push

Status: **Next**

- Add remote only after explicit user authorization.
- Push only after explicit user authorization.
- Complete GitHub About description, topics, and profile pin.

## Design principles

- Local-first before integration-heavy.
- Explainable rules before probabilistic analysis.
- Fictional data in public artifacts.
- Human review before operational action.
- Small, testable increments.
